# OpenCode 十節課教案專案

## 專案概述

這是一個 AI 助教 OpenCode 的國中資訊科技教案專案（適用七～九年級），共 10 節課，每節 45 分鐘。專案包含四份「懶人包」操作指南（#00 環境建置、#01 連接 NotebookLM、#02 連接 GitHub、#08 FB 頻道摘要 Skill）、三份規劃中懶人包（#03、#04、#07），以及三份教案文件。

## 檔案命名慣例

- **懶人包**：`{編號}-{名稱}.md`（如 `00-環境建置.md`）
- **教案**：`OpenCode-十節課教案-{角色}.md`（角色：教師大綱 / 教師手冊 / 學生講義）

## 核心檔案關係

| 懶人包 | 對應課次 | 用途 |
|--------|---------|------|
| #00 環境建置 | 第 4–5 課 | 安裝 Git、gh、uv、winget |
| #01 連接 NotebookLM | 第 6–8 課 | 安裝 nlm、登入 Google、MCP 設定 |
| #08 FB 頻道摘要 Skill | 第 9 課 | 安裝 SKILL.md、測試互動 |

## 教案架構

- **教師大綱**：每課的教學目標、時間分配、評量方式總表
- **教師手冊**：完整教學引導（含老師引導語、卡關處理、評量方式）
- **學生講義**：學生用的活動手冊（含檢查點、記錄欄位、完成確認）

## 每課格式慣例

每課包含：
- 教學目標（以 `- [ ]` 條列）
- 時間分配表（教師大綱/教師手冊）
- 引導語（以 `>` 標記，教師手冊）
- 卡關處理表（教師手冊）
- 檢查點（以 `□` 標記，學生講義）

## 懶人包格式慣例

每份懶人包包含：
- 前置中繼資料（title, date, type, version, status, tags）
- 版本說明
- 先備條件檢查清單
- 步驟零：環境檢查（`> 🔁 重複執行檢查` 模式）
- 後續步驟（操作指令 + 注意事項）
- 復原機制（「如果安裝失敗，如何重來」）
- 常見問題表格
- 更新紀錄
- 相關連結

## 技術棧

- **OS**: Windows 10+（主要目標平台），支援 macOS/Linux
- **Shell**: PowerShell 5.1（Windows），bash（macOS/Linux）
- **工具**: Git, GitHub CLI (gh), uv, winget
- **MCP**: notebooklm-mcp-cli (nlm)
- **AI 平台**: OpenCode, OpenRouter (API Key), Google NotebookLM
