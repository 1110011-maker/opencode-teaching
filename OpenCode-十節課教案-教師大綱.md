# AI 助教 OpenCode — 十節課教案（教師大綱）

> 適用對象：國中七～九年級電腦 / 資訊科技課
> 每節課：45 分鐘 | 學生自備電腦（可自行安裝軟體）
> 授課教師：科技領域教師
> 對應懶人包：#00 環境建置、#01 連接 NotebookLM、#08 安裝 FB 頻道摘要 Skill

---

| 課次 | 主題 | 核心懶人包 | 重點 |
|------|------|-----------|------|
| 1 | AI 與 OpenCode 初體驗 | — | 認識 AI、觀看 OpenCode 示範 |
| 2 | 安裝 OpenCode 與第一次對話 | — | 下載桌面版、申請 API Key、初次對話 |
| 3 | 提示技巧入門 | — | 好問題 vs 壞問題、三種實戰練習 |
| 4 | 懶人包 #00 環境建置（上） | #00 | 步驟零環境檢查 + 步驟一安裝 Git |
| 5 | 懶人包 #00 環境建置（下） | #00 | 步驟二 gh + 步驟三 uv + 步驟四驗證 |

## 第一課：AI 與 OpenCode 初體驗

**教學目標：**
- 能說出 AI 的基本定義與生活中的應用實例
- 能理解 OpenCode 是什麼以及它能做什麼
- 能對 AI 工具產生興趣與正向態度

**時間分配：**

| 時間 | 活動 | 說明 |
|------|------|------|
| 0'–5' | 暖身討論 | 老師提問：「你聽過或用過哪些 AI？」 |
| 5'–15' | AI 是什麼？ | 簡單定義 + 生活案例（手機、YT、Gmail、ChatGPT） |
| 15'–25' | OpenCode 是什麼？ | 介紹：AI 程式設計助教，能幫你寫程式、整理筆記、做報告 |
| 25'–40' | 現場示範 | 老師開啟 OpenCode，展示：講解程式、整理文章、產生檔案 |
| 40'–45' | 本課總結 | 回顧重點 + 預告下週要自己安裝 OpenCode |

**評量方式：** 口頭問答、課堂參與度

---

## 第二課：安裝 OpenCode 與第一次對話

**教學目標：**
- 能自行下載並安裝 OpenCode 桌面版
- 能取得 API Key 並完成基本設定
- 能與 OpenCode 進行第一次對話

**時間分配：**

| 時間 | 活動 | 說明 |
|------|------|------|
| 0'–5' | 回顧與本課目標 | 上週看了示範，今天輪到你安裝！ |
| 5'–20' | 下載安裝 OpenCode | 官網下載 → 安裝 → 開啟 |
| 20'–30' | 申請 API Key | 註冊 OpenRouter → 建立 API Key → 貼入 OpenCode |
| 30'–40' | 第一次對話 | 自由發問：「Hello OpenCode」「1+1=？」「幫我解釋 for 迴圈」 |
| 40'–45' | 總結 | 分享你問了什麼 + 下週預告 |

**評量方式：** 完成安裝並成功對話（老師巡視確認）

---

## 第三課：提示技巧入門

**教學目標：**
- 能分辨好問題與壞問題的差異
- 能寫出具體、清楚的提示詞（Prompt）
- 能應用提示技巧完成三種任務

**時間分配：**

| 時間 | 活動 | 說明 |
|------|------|------|
| 0'–5' | 回顧 | 上週你問了什麼？回答滿意嗎？ |
| 5'–15' | 好問題 vs 壞問題 | 範例比較：含糊 vs 具體 |
| 15'–25' | 練習一：作業輔助 | 請 OpenCode 解釋一個觀念 |
| 25'–35' | 練習二：資料整理 | 請 OpenCode 整理重點 |
| 35'–40' | 練習三：創意發想 | 請 OpenCode 給靈感 |
| 40'–45' | 總結 | 好問題的三個原則 |

**評量方式：** 繳交三組對話紀錄截圖

## 第四課：懶人包 #00 環境建置（上）

**教學目標：**
- 能理解為什麼需要安裝 Git、gh、uv
- 能執行環境檢查，判斷哪些工具已安裝
- 能安裝 Git for Windows

**時間分配：**

| 時間 | 活動 | 說明 |
|------|------|------|
| 0'–5' | 為什麼要裝這些工具？ | 說明 Git(版本控管)、gh(GitHub 溝通)、uv(Python 工具) 的角色 |
| 5'–20' | 步驟零：環境檢查 | 依懶人包檢查 OS、網路、Git、gh、uv、winget |
| 20'–25' | 步驟零之一：安裝 winget | 如有需要，依懶人包步驟安裝 |
| 25'–40' | 步驟一：安裝 Git | 依懶人包指令安裝並確認版本 |
| 40'–45' | 總結與進度確認 | 記錄哪些裝好了、哪些下週繼續 |

**評量方式：** 完成環境檢查表（老師發學習單或數位表單）

## 第五課：懶人包 #00 環境建置（下）

**教學目標：**
- 能安裝 GitHub CLI（gh）
- 能安裝 uv
- 能執行最終驗證，確認四種工具皆正常運作

**時間分配：**

| 時間 | 活動 | 說明 |
|------|------|------|
| 0'–3' | 回顧 | 上週裝了 Git，本週完成剩下的工具 |
| 3'–15' | 步驟二：安裝 GitHub CLI | 依懶人包指令安裝並確認版本 |
| 15'–27' | 步驟三：安裝 uv | 依懶人包指令安裝並確認版本 |
| 27'–40' | 步驟四：最終驗證 | git --version / gh --version / uv --version / winget --version |
| 40'–45' | 懶人包 #00 完成！ | 四項工具全部確認 ✅ 預告下週 NotebookLM |

**評量方式：** 繳交最終驗證的指令執行截圖（四項版本號）

## 第六課：懶人包 #01 連接 NotebookLM（上）

**教學目標：**
- 能說明 NotebookLM 是什麼以及三角關係原理
- 能理解 MCP 的基本概念（翻譯官比喻）
- 能安裝 notebooklm-mcp-cli

**時間分配：**

| 時間 | 活動 | 說明 |
|------|------|------|
| 0'–5' | 什麼是 NotebookLM？ | Google 的 AI 研究助理，以你的資料為基礎 |
| 5'–15' | 三角關係與 MCP 原理 | OpenCode ← MCP → nlm ← 登入 → NotebookLM |
| 15'–25' | 步驟零：環境檢查 | uv 已安裝？網路？OpenCode 版本？ |
| 25'–40' | 步驟一：安裝 nlm | uv tool install notebooklm-mcp-cli |
| 40'–45' | 總結 | nlm --version 確認 |

**評量方式：** 口頭說明三角關係 + nlm 安裝完成確認

---

## 第七課：懶人包 #01 連接 NotebookLM（下）

**教學目標：**
- 能完成 Google 帳號登入認證
- 能設定 MCP 連接
- 能建立本地資料夾並驗證連接成功

**時間分配：**

| 時間 | 活動 | 說明 |
|------|------|------|
| 0'–3' | 回顧 | 上週裝了 nlm，今天要連上 NotebookLM |
| 3'–15' | 步驟二：登入 Google 帳號 | nlm login → 瀏覽器授權 → nlm doctor 確認 |
| 15'–25' | 步驟三：MCP 連接設定 | nlm setup add opencode → nlm setup list 確認 |
| 25'–30' | 步驟四：建立本地資料夾 | 建立 slides/ infographics/ audio/ 等目錄 |
| 30'–40' | 步驟五～六：重啟與測試 | 重啟 OpenCode → 建立測試 notebook → 刪除 |
| 40'–45' | 懶人包 #01 完成！ | ✅ 確認連接成功 |

**評量方式：** 測試 notebook 建立成功之螢幕截圖

## 第八課：NotebookLM 實戰應用

**教學目標：**
- 能建立 notebook 並上傳資料來源
- 能產出 slide deck、mind map、infographic
- 能下載產出結果到本地資料夾

**時間分配：**

| 時間 | 活動 | 說明 |
|------|------|------|
| 0'–3' | 回顧三角關係 | 快速複習 OpenCode → nlm → NotebookLM |
| 3'–13' | 建立 notebook + 上傳資料 | 選擇主題（課本內容/時事/興趣），上傳為來源 |
| 13'–23' | 產出簡報 | studio_create slide_deck → 下載 PDF |
| 23'–30' | 產出心智圖 + 資訊圖表 | mind_map + infographic |
| 30'–42' | 自由探索 | 學生自行嘗試不同產出組合 |
| 42'–45' | 總結 | 你最喜歡哪種產出？ |

**評量方式：** 繳交一份自選主題的產出（簡報/心智圖/資訊圖表擇一）

---

## 第九課：懶人包 #08 安裝 FB 頻道摘要 Skill

**教學目標：**
- 能理解什麼是 OpenCode Skill（擴充功能）
- 能完成 fb-notebooklm Skill 的安裝
- 能使用「FB 摘要」指令進行互動式產出

**時間分配：**

| 時間 | 活動 | 說明 |
|------|------|------|
| 0'–5' | 什麼是 Skill？ | 像手機 App 一樣擴充 OpenCode 的能力 |
| 5'–10' | 步驟一：確認認證 | nlm doctor 確認連線正常 |
| 10'–25' | 步驟二：安裝 Skill 檔案 | 建立技能資料夾 → 寫入 SKILL.md → 確認檔案存在 |
| 25'–30' | 步驟三：重啟 OpenCode | 完全關閉再開啟 |
| 30'–42' | 步驟四：測試 Skill | 說「FB 摘要」→ 依互動流程選擇 → 確認正常運作 |
| 42'–45' | 懶人包 #08 完成！ | ✅ 總結三包都裝好了 |

**評量方式：** Skill 安裝完成並成功載入（老師巡視確認）

## 第十課：整合應用 + 成果展示

**教學目標：**
- 能串聯 #00、#01、#08 完成一個完整專題
- 能展示自己的 AI 輔助學習成果
- 能討論 AI 使用時的注意事項與倫理議題

**時間分配：**

| 時間 | 活動 | 說明 |
|------|------|------|
| 0'–5' | 完整工作流程回顧 | #00 建環境 → #01 連 NotebookLM → #08 裝 Skill → 開始用！ |
| 5'–10' | 選題與規劃 | 自選主題 → 決定使用哪些工具 → 規劃步驟 |
| 10'–30' | 實作時間 | 學生獨立完成，老師巡迴協助 |
| 30'–40' | 成果分享 | 3–4 位學生展示（用了什麼工具？產出什麼？） |
| 40'–45' | AI 素養 + 課程總結 | 何時該用 AI、何時不該用、資料驗證的重要性 |

**評量方式：** 完成一份專題產出 + 簡短心得（口頭或書面）
