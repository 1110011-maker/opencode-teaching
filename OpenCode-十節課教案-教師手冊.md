# AI 助教 OpenCode — 十節課教師手冊

> 適用對象：國中七～九年級電腦 / 資訊科技課
> 每節課：45 分鐘 | 學生可自行安裝軟體
> 作者：科技領域教師
> 對應懶人包：#00 環境建置、#01 連接 NotebookLM、#08 安裝 FB 頻道摘要 Skill

---

## 總則

### 課前準備（通用）

| 項目 | 說明 |
|------|------|
| 教材檔案 | 將 `00-環境建置.md`、`01-連接-NotebookLM.md`、`08-安裝FB頻道摘要Skill.md` 三份懶人包準備好，可於課堂投影或發給學生 |
| 教師電腦 | 請教師在上課前完整跑過一遍三個懶人包，確認所有指令在學校電腦環境中可正常執行 |
| 網路環境 | 確認教室網路暢通，可連線至 github.com、opencode.ai、openrouter.ai、google.com |
| 投影設備 | 建議每堂課都用投影示範操作 |
| 學生講義 | 印出對應課次的學生講義頁面，或提供電子檔 |

### 通用卡關處理原則

| 卡關狀況 | 處理方式 |
|----------|---------|
| 指令打錯字 | 請學生仔細看螢幕上的錯誤訊息，通常是少了連字號或空格 |
| 權限不足 | 以系統管理員身分執行 PowerShell（右鍵 → 以系統管理員身分執行） |
| 安裝卡住 | 等候 2–3 分鐘，若無回應請關閉 PowerShell 重來 |
| 找不到指令 | 關閉 PowerShell 重新開啟再試 |
| 學生進度落後 | 請已完成同學協助鄰座同學（同儕教學），或請學生於課後補做 |

## 第一課：AI 與 OpenCode 初體驗

### 備課提醒

- 準備投影機，教師電腦需已安裝 OpenCode 並可正常運作
- 課前先想好 2–3 個現場示範的情境（例：解釋 for 迴圈、整理一段文章、寫一個計算機程式）
- 可準備一段「壞示範」（故意問一個模糊的問題）來對比好問題

### 教學目標

- 學生能說出 AI 的基本定義與生活實例
- 學生能理解 OpenCode 的功能與定位
- 學生對 AI 工具產生正向態度

### 教學流程

**0'–5' 暖身討論**

> 老師引導語：「同學們，你們聽過 AI 嗎？舉手讓我知道一下。好，很多人舉手。那你們用過哪些 AI 產品？跟旁邊的同學討論一分鐘，我待會請幾位來分享。」

- 讓學生跟鄰座討論 1 分鐘
- 點 2–3 位學生分享
- 總結學生的答案，補充常見但學生沒提到的例子（推薦影片、垃圾郵件過濾）

**5'–15' AI 是什麼？**

> 老師引導語：「AI 這個詞大家常聽到，但到底什麼是 AI？簡單說，AI 就是『讓電腦像人一樣思考、學習、解決問題的技術』。」

- 用投影展示簡報或板書：
  - AI = Artificial Intelligence
  - 生活中的 AI（手機解鎖、YT 推薦、ChatGPT、自動駕駛）
- 強調「AI 能做 vs 不能做」：
  - 能做：理解語言、生成文字、分析資料、解決問題
  - 不能做：沒有自己的想法、可能會出錯、不能代替你思考

> 老師引導語：「最重要的一句話：AI 是輔助工具，不是作弊工具。用 AI 幫助你學習很好，但直接複製貼上交作業就不對了。」

**15'–25' OpenCode 是什麼？**

> 老師引導語：「今天要介紹一個很厲害的 AI 工具叫 OpenCode。它是什麼呢？它是一個 AI 程式設計助教！」

- 說明 OpenCode 能做什麼：
  - 解釋程式碼
  - 幫你寫程式
  - 找程式錯誤
  - 整理資料
  - 操作其他工具（預告 NotebookLM）

> 老師引導語：「想像一下，你有一個 24 小時不下班、超有耐心的同學坐在你旁邊，隨時可以問他問題。這就是 OpenCode。」

**25'–40' 現場示範**

> 老師引導語：「現在我來實際操作給你們看。注意看螢幕，我問 OpenCode 幾個問題，看看它怎麼回答。」

- 示範 1：解釋程式碼
  - 輸入：「請用國中生聽得懂的話解釋這段程式在做什麼：for i in range(10): print(i)」
  - 讓學生看 OpenCode 的回答

- 示範 2：整理資料
  - 輸入一段文章，請 OpenCode 整理重點
  - 例如貼上臺灣溫泉的文章，說「幫我用三點條列式整理重點」

- 示範 3：產生檔案
  - 輸入：「幫我寫一個 Python 計算機程式，可以讓使用者輸入兩個數字，然後選擇要加減乘除，最後輸出結果」

> 老師引導語：「看到了嗎？我只要用中文跟它說，它就能做到。下一節課，換你們自己來安裝和試試看！」

**40'–45' 總結**

> 老師引導語：「今天我們學了三件事：第一，AI 是什麼？第二，OpenCode 能做什麼？第三，下週你們要自己安裝 OpenCode 來玩！」

### 學生常見卡關

| 卡關 | 因應方式 |
|------|---------|
| 學生對 AI 已有誤解（覺得 AI=作弊） | 說明工具的善用 vs 濫用，舉例：計算機不是作弊，但考試偷用計算機才是 |
| 學生覺得 AI 離自己很遠 | 舉貼近生活的例子：手機解鎖、YT 推薦、Google 翻譯 |

### 評量方式

口頭問答、課堂參與度。老師可隨機點名確認學生有在聽。

## 第二課：安裝 OpenCode 與第一次對話

### 備課提醒

- 課前確認官網 https://opencode.ai 可正常連線
- 課前確認 https://openrouter.ai 可正常連線
- 教師自己先準備一組 API Key，以備學生申請失敗時借用（但建議讓學生自己申請）
- 提醒學生帶自己的筆電（如使用學校電腦，需確認安裝權限）
- 可以準備一個 USB 隨身碟裝好安裝檔，以免官網下載太慢

### 教學目標

- 學生能下載並安裝 OpenCode 桌面版
- 學生能取得 OpenRouter API Key
- 學生能與 OpenCode 進行第一次對話

### 教學流程

**0'–5' 回顧與本課目標**

> 老師引導語：「上週我示範了 OpenCode，這週輪到你們自己安裝了！今天的目標很簡單：裝好 OpenCode、拿到 API Key、跟它說第一句話。」

**5'–20' 下載安裝 OpenCode**

> 老師引導語：「第一步，打開瀏覽器，輸入 opencode.ai，然後點 Download。」

- 用投影同步示範每個步驟
- 學生操作時巡視教室確認進度

> 老師引導語（遇到下載慢時）：「官網下載如果太慢，我這邊有準備安裝檔，可以來我這邊複製。」

- 安裝完成後，請學生開啟 OpenCode

> 老師引導語：「安裝完成的人舉手。還沒裝好的沒關係，旁邊同學幫他看一下。」

**20'–30' 申請 API Key**

> 老師引導語：「OpenCode 需要一個『API Key』才能工作。想像 API Key 是一把鑰匙，讓 OpenCode 可以打開 AI 模型的大門。」

- 用投影示範：前往 openrouter.ai → 註冊 → 建立 Key
- 提醒學生：
  - Key 要複製並貼到 OpenCode 的設定中
  - **API Key 不要傳給別人！**

> 老師引導語：「API Key 就像你的密碼，不要截圖傳給同學，也不要貼在網路上。如果有人拿到你的 Key，他就可以用你的名義使用 AI。」

**30'–40' 第一次對話**

> 老師引導語：「現在 OpenCode 裝好了，Key 也貼上去了，來跟它說第一句話吧！」

- 請學生輸入：「Hello OpenCode！我是（名字），請多多指教！」
- 再試：「1+1 等於多少？」
- 最後：「請用簡單的話解釋什麼是演算法」

> 老師引導語：「把你問的問題和 OpenCode 的回答記下來，這是今天的作業。」

**40'–45' 總結**

> 老師引導語：「舉手讓我知道：誰成功跟 OpenCode 對話了？（統計人數）非常好！下週我們要學怎麼問問題才能得到最好的答案。」

### 學生常見卡關

| 卡關 | 解法 |
|------|------|
| API Key 貼錯位置 | 確認 Key 貼在 OpenCode 設定的 Provider 欄位，且沒有多餘空格 |
| OpenCode 一直說連線失敗 | 確認 API Key 是否有效（到 OpenRouter 網站檢查 Key 狀態） |
| 下載安裝檔被防毒軟體擋 | 暫時關閉防毒軟體，或加入例外清單 |
| OpenCode 安裝後無法開啟 | 重新安裝，或檢查是否為 Windows 版本 |

### 評量方式

老師巡視教室，確認每位學生的 OpenCode 已安裝完成並可正常對話。完成者於講義上簽名。

## 第三課：提示技巧入門

### 備課提醒

- 準備 3–4 組好問題/壞問題的對照範例
- 課前選定一篇短文（可用國文課本、任何知識型文章）作為練習素材
- 建議老師先想好自己的「壞示範」，讓學生感受差異

### 教學目標

- 學生能分辨好問題與壞問題的差異
- 學生能寫出具體、清楚的提示詞
- 學生能應用提示技巧完成三種任務

### 教學流程

**0'–5' 回顧**

> 老師引導語：「上週你們都安裝好 OpenCode 了，回家有繼續玩嗎？你問了它什麼問題？回答滿意嗎？」

- 請 2–3 位學生分享

**5'–15' 好問題 vs 壞問題**

> 老師引導語：「同樣的問題，用不同方式問，AI 的回答可以差很多。我來示範給你看。」

**壞示範（投影輸入）：**
```
幫我寫程式
```
> 老師引導語：「看，OpenCode 不知道你要寫什麼，只能問你更多問題。」

**好示範（投影輸入）：**
```
幫我用 Python 寫一個加法計算機，使用者輸入兩個數字，程式輸出相加的結果
```
> 老師引導語：「差很多吧？好問題的關鍵是：具體、有格式、有範圍。」

- 介紹好問題公式：**角色 + 任務 + 細節 + 格式**
- 舉例：
  - 角色：「你是國中老師」
  - 任務：「解釋光合作用」
  - 細節：「用生活例子」
  - 格式：「用條列式」

**15'–25' 練習一：作業輔助**

> 老師引導語：「現在換你們試。想一個你最近上課聽不懂的觀念，不管是數學、自然、社會都可以，用剛剛學到的方法問 OpenCode。」

- 學生操作，老師巡視
- 提醒學生不要直接說「解釋這個」，要說清楚是什麼觀念

**25'–35' 練習二：資料整理**

> 老師引導語：「第二個練習，我投影一段文章，請 OpenCode 幫你整理重點。要用條列式，每一點不超過 20 個字。」

- 投影文章（如臺灣溫泉或當週國文/社會課內容）
- 學生操作

**35'–40' 練習三：創意發想**

> 老師引導語：「第三個練習，請 OpenCode 幫你想靈感。例如你想做一個專題報告但沒有想法，問問 OpenCode 有什麼建議。」

**40'–45' 總結**

> 老師引導語：「今天的三個重點記下來：第一，問題要具體；第二，要給格式要求；第三，要給範圍限制。記住這三個原則，你就能得到更好的答案。」

### 學生常見卡關

| 卡關 | 解法 |
|------|------|
| 問題還是太模糊 | 帶學生用公式套：角色+任務+細節+格式 |
| 學生直接複製貼上作業 | 說明這不是練習的目的，重點是學會如何問問題 |
| OpenCode 回答太長看不懂 | 要求「請用國中生聽得懂的話回答」或「限制在 100 字以內」 |

### 評量方式

繳交三組對話紀錄截圖或書面記錄（於講義中書寫即可）。

## 第四課：懶人包 #00 環境建置（上）

### 備課提醒

- 教師課前務必在自己電腦完整跑過一次 #00 懶人包的所有步驟
- 準備一個環境檢查表（可投影）讓學生對照
- 注意：部分學校電腦有還原系統，安裝後重開機可能會消失。如有此情況，建議改以教師機統一安裝或提前跟資訊組確認

### 教學目標

- 學生能理解 Git、gh、uv 各自的功能
- 學生能執行環境檢查並判讀結果
- 學生能安裝 Git for Windows

### 教學流程

**0'–5' 為什麼要裝這些工具？**

> 老師引導語：「今天開始我們要進入一個新階段——安裝工具。為什麼要裝這些東西？因為未來 OpenCode 要透過這些工具去做更多厲害的事。」

- Git → 版本的時光機，記錄每一次修改
- gh → GitHub 的遙控器，跟雲端溝通
- uv → Python 的 App Store，安裝套件用

> 老師引導語：「你可能覺得這些東西聽起來很難，但不用擔心，照著步驟做就好。而且這些工具是業界工程師每天在用的，你現在學會，未來一定用得到。」

**5'–20' 步驟零：環境檢查**

> 老師引導語：「第一步不是直接安裝，而是先檢查你的電腦已經有什麼了。打開 PowerShell，我投影寫了什麼，你們就打什麼。」

- 打開 PowerShell 方法：Win+R → 輸入 `powershell` → Enter
- 依序執行懶人包 #00 的 6 項檢查

> 老師引導語：「把你檢查的結果寫在講義的表格上。有打勾的表示已經裝好了，打叉的等等或下週會補裝。」

**20'–25' 步驟零之一：安裝 winget（如有需要）**

> 老師引導語：「winget 是 Windows 的套件管理員，就像手機的 App Store。等一下裝 Git 會用到它。如果你的電腦沒有 winget，我們來裝。」

- 這個步驟較複雜，建議老師統一處理或指派已完成學生協助

**25'–40' 步驟一：安裝 Git**

> 老師引導語：「現在來安裝 Git。還記得 Git 是什麼嗎？對，版本的時光機。請在 PowerShell 輸入我投影上的指令。」

- 投影指令：
```powershell
winget install --id Git.Git --accept-source-agreements --accept-package-agreements
```

> 老師引導語：「安裝需要一點時間，請耐心等候。裝好後輸入 git --version 確認。」

**40'–45' 總結與進度確認**

> 老師引導語：「下課前確認一下今天的進度：Git 裝好的舉手。（統計）好，winget 沒裝到的下課可以來找我。下週我們裝 gh 和 uv，完成環境建置！」

### 學生常見卡關

| 卡關 | 解法 |
|------|------|
| PowerShell 找不到指令 | 確認沒有打錯字，指令有空格的地方要有空格 |
| winget 安裝失敗 | 確認 Windows 版本（需 Windows 10 1709+），改用離線安裝包 |
| git --version 無回應 | 關掉 PowerShell 重新開啟，或重開機 |
| 安裝過程卡住很久 | 關掉視窗重來，確認網路連線正常 |

### 評量方式

環境檢查表填寫完成（講義上的表格），老師巡視確認。

## 第五課：懶人包 #00 環境建置（下）

### 備課提醒

- 確認上週 Git 安裝的進度，部分學生可能需要補裝
- 這堂課指令較多，建議投影一步一步帶
- 提醒學生四個工具全部裝好後要截圖或記錄版本號

### 教學目標

- 學生能安裝 GitHub CLI（gh）
- 學生能安裝 uv
- 學生能執行最終驗證，確認四項工具皆正常

### 教學流程

**0'–3' 回顧**

> 老師引導語：「上週我們裝了 Git，這週要完成剩下的 gh 和 uv。上週 Git 還沒裝好的同學，請舉手，我會請旁邊同學先幫你看一下。」

**3'–15' 步驟二：安裝 GitHub CLI**

> 老師引導語：「GitHub CLI，簡稱 gh，是讓你的電腦跟 GitHub 雲端倉庫溝通的工具。想像它是一個遙控器，可以操控 GitHub 上的東西。」

- 投影指令：
```powershell
winget install --id GitHub.cli --accept-source-agreements --accept-package-agreements
```

> 老師引導語：「安裝完成後，輸入 gh --version 確認。」

**15'–27' 步驟三：安裝 uv**

> 老師引導語：「uv 是 Python 的套件管理器。下週我們要安裝一個叫 nlm 的工具，就會用到 uv。請輸入投影上的指令。」

- 投影指令：
```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

> 老師引導語：「安裝完成後，關掉 PowerShell 再重新打開，然後輸入 uv --version 確認。」

**27'–40' 步驟四：最終驗證**

> 老師引導語：「現在來驗證四個工具全部都可以正常使用。依序輸入這四個指令，把版本號寫在講義上。」

```powershell
git --version
gh --version
uv --version
winget --version
```

> 老師引導語：「四個都有顯示版本號的舉手！恭喜，你的環境建置完成了！」

**40'–45' 懶人包 #00 完成！**

> 老師引導語：「給自己一個掌聲！你們完成了第一份懶人包的安裝。這些工具是專業工程師在用的，你們已經比很多大人還厲害了。下週我們要連接 NotebookLM！」

### 學生常見卡關

| 卡關 | 解法 |

| gh 安裝後指令找不到 | 關閉 PowerShell 重開 |

## 第六課：懶人包 #01 連接 NotebookLM（上）

### 備課提醒

- 教師課前先跑過一次 #01 懶人包確認流程
- 準備 NotebookLM 的三角關係圖（可事先畫在白板或投影）
- 提醒學生這週需要 Google 帳號（沒有的請事先申請或跟老師說）

### 教學目標

- 學生能說明 NotebookLM 是什麼
- 學生能說出三角關係中 OpenCode、nlm、NotebookLM 的角色
- 學生能安裝 notebooklm-mcp-cli（nlm）

### 教學流程

**0'–5' 什麼是 NotebookLM？**

> 老師引導語：「今天我們要認識一個新的 AI 工具——NotebookLM。它是 Google 推出的 AI 研究助理。跟 ChatGPT 最大的不同是：ChatGPT 根據它學過的資料來回答，而 NotebookLM 只根據『你給它的資料』來回答。」

- 比喻：NotebookLM 像一個「只讀你指定的課本」的超級助教

**5'–15' 三角關係與 MCP 原理**

> 老師引導語：「現在我來說明 OpenCode 跟 NotebookLM 是怎麼連接的。請看這個三角關係圖。」

- 投影三角關係：
```
你 ← 講中文 → OpenCode ← MCP 協定 → nlm ← 登入 → NotebookLM
```

> 老師引導語：「OpenCode 是你的 AI 助教，它聽得懂中文。但 OpenCode 沒辦法直接操作 NotebookLM，因為 NotebookLM 沒有官方的程式接口。所以中間需要一個『翻譯官』——就是 nlm。」

> 老師引導語：「nlm 做的事情很酷：它假裝是人在操作瀏覽器，幫你自動點擊 NotebookLM 的網頁功能。而 MCP 是 OpenCode 跟 nlm 溝通的標準語言。」

**15'–25' 步驟零：環境檢查**

> 老師引導語：「在裝 nlm 之前，先確認三件事：第一，上週裝的 uv 能不能用？第二，網路通不通？第三，OpenCode 版本對不對？」

- 學生在 PowerShell 中依序執行：

```powershell
uv --version
ping google.com
opencode --version
```

**25'–40' 步驟一：安裝 nlm**

> 老師引導語：「現在來裝 nlm。因為上週我們裝了 uv，所以現在可以用 uv 來安裝 nlm。先檢查是不是已經裝過了。」

```powershell
uv tool list
```

> 老師引導語：「如果清單中有 notebooklm-mcp-cli，表示之前已經裝過了。如果沒有，執行安裝。」

```powershell
uv tool install notebooklm-mcp-cli
```

> 老師引導語：「安裝需要一點時間，請稍等。裝好後輸入 nlm --version 確認。」

**40'–45' 總結**

> 老師引導語：「今天的進度：nlm 安裝完成了！下週我們要做三件事：登入 Google 帳號、設定 MCP 連接、測試連線。全部完成後，你的 OpenCode 就可以操作 NotebookLM 了！」

### 學生常見卡關

| 卡關 | 解法 |
|------|------|
| uv tool install 失敗 | 檢查網路連線，確認 uv 版本為最新 |
| nlm --version 找不到 | 關閉 PowerShell 重開 |
| 不懂三角關係 | 用實體比喻：OpenCode 是老闆，nlm 是助理，NotebookLM 是工廠 |
| 學生沒有 Google 帳號 | 請學生課後申請，或使用學校提供的 Google 教育帳號 |

### 評量方式

口頭抽問三角關係，nlm 安裝完成確認。

## 第七課：懶人包 #01 連接 NotebookLM（下）

### 備課提醒

- 這堂課需要學生登入 Google 帳號，課前提醒學生記得帳號密碼
- 登入過程需要瀏覽器授權，課前確認教室電腦的瀏覽器可正常開啟 Google 登入頁面
- nlm login 會開啟瀏覽器，確認教室電腦沒有被封鎖彈出視窗

### 教學目標

- 學生能完成 Google 帳號登入認證
- 學生能設定 MCP 連接
- 學生能建立本地資料夾並測試連線成功

### 教學流程

**0'–3' 回顧**

> 老師引導語：「上週我們裝了 nlm，今天要讓它真正連上 NotebookLM。總共要做四件事：登入 Google、設定 MCP、建資料夾、測試。」

**3'–15' 步驟二：登入 Google 帳號**

> 老師引導語：「第一步，先用 nlm doctor 檢查是不是已經登入過了。」

```powershell
nlm doctor
```

> 老師引導語（如果已認證）：「已經登入過了，直接跳到步驟三。」
> 老師引導語（如果未認證）：「請輸入 nlm login，瀏覽器會自動開啟 Google 登入頁面。」

```powershell
nlm login
```

> 老師引導語：「在瀏覽器中選擇你的 Google 帳號，輸入密碼，然後按允許。完成後回到 PowerShell，輸入 nlm doctor 確認。」

> ⚠️ 提醒學生：這一步需要手動在瀏覽器操作，不是自動完成的。

**15'–25' 步驟三：MCP 連接設定**

> 老師引導語：「現在要讓 OpenCode 知道 nlm 的存在，把它們兩個連接起來。」

```powershell
nlm setup list
```

> 老師引導語：「如果清單中有 opencode，表示已經設定過了。如果沒有，執行：」

```powershell
nlm setup add opencode
```

> 老師引導語：「設定完成後再用 nlm setup list 確認，應該會看到 opencode 在清單中。」

**25'–30' 步驟四：建立本地資料夾**

> 老師引導語：「我們要在電腦中建立一些資料夾，以後 NotebookLM 產出的檔案會自動下載到這裡。」

- 投影指令，讓學生複製整段貼上：

```powershell
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\Documents\NotebookLM\slides"
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\Documents\NotebookLM\infographics"
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\Documents\NotebookLM\audio"
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\Documents\NotebookLM\video"
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\Documents\NotebookLM\docs"
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\Documents\NotebookLM\sheets"
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\Documents\NotebookLM\mindmaps"
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\Documents\NotebookLM\quizzes"
```

**30'–40' 步驟五～六：重啟 OpenCode 並測試**

> 老師引導語：「現在完全關閉 OpenCode，再重新打開。」

> 老師引導語：「打開後，在對話框中輸入：『請幫我建立一個名為「測試筆記本」的 NotebookLM notebook』」

- 如果成功 → OpenCode 會回傳 notebook ID
- 再輸入：「請刪除剛剛建立的測試筆記本」

> 老師引導語：「如果兩個指令都成功執行，恭喜你——NotebookLM 連接成功了！」

**40'–45' 懶人包 #01 完成！**

> 老師引導語：「給自己鼓掌！你完成了第二份懶人包。現在你的 OpenCode 可以操控 NotebookLM 了。下週我們要實際用它來做簡報、心智圖！」

### 學生常見卡關

| 卡關 | 解法 |
|------|------|
| nlm login 瀏覽器沒開 | 手動開啟瀏覽器前往 google.com 登入後，再執行一次 nlm login |
| 登入後 nlm doctor 仍顯示未認證 | 重跑 nlm login，確認有完成瀏覽器授權 |
| nlm setup add opencode 失敗 | 確認 nlm 版本夠新（nlm --version），更新：uv tool upgrade notebooklm-mcp-cli |
| OpenCode 看不到 NotebookLM 工具 | 確認有設定 MCP 且已完全重啟 OpenCode |
| 學生沒有 Google 帳號 | 跟同學借用或使用學校 Google 教育帳號 |

### 評量方式

測試 notebook 建立成功並刪除的截圖。

## 第八課：NotebookLM 實戰應用

### 備課提醒

- 課前準備 2–3 個範例主題供學生參考
- 確認 nlm doctor 認證正常
- 這堂課時間較緊湊，建議老師示範一個主題，學生再自己試

### 教學目標

- 學生能建立 notebook 並上傳資料
- 學生能產出 slide deck 並下載
- 學生能產出 mind map 和 infographic

### 教學流程

**0'–3' 回顧三角關係**

> 老師引導語：「快速複習：你對 OpenCode 說中文，OpenCode 透過 nlm 叫 NotebookLM 做事，成品回到你電腦。今天我們要實際來感受這個流程！」

**3'–13' 建立 notebook + 上傳資料**

> 老師引導語：「第一步，選一個你有興趣的主題。我建議可以選自然課或社會課的某一章內容，或是任何你想了解的知識。」

- 對 OpenCode 說：「請幫我建立一個名為『（主題）』的 NotebookLM notebook」
- 準備資料：可以是課本內容、網路文章、或自己的筆記

> 老師引導語：「把你準備的資料貼給 OpenCode，請它加入 notebook。」

**13'–23' 產出簡報**

> 老師引導語：「資料上傳後，請 OpenCode 幫你產出簡報。」

- 對 OpenCode 說：「請幫我從這個 notebook 產生一份 slide deck 簡報，語言用繁體中文」

> 老師引導語：「簡報產出需要等 1–2 分鐘。下載後打開看看，是不是比你用手做快很多？」

**23'–30' 產出心智圖 + 資訊圖表**

> 老師引導語：「接下來試試心智圖和資訊圖表。」

- 心智圖：「請幫我從這個 notebook 產生一份 mind map」
- 資訊圖表：「請幫我從這個 notebook 產生一張 infographic，方向用橫向，繁體中文」

**30'–42' 自由探索**

> 老師引導語：「現在是自由時間。你可以試試其他產出方式：測驗題、閃卡、報告。也可以換一個主題重新試一次。把最滿意的結果記錄下來。」

**42'–45' 總結**

> 老師引導語：「今天你學會了：讓 AI 幫你做簡報、心智圖、資訊圖表。但記住，AI 產出的東西要自己看過、修改過，才是你真正的作品。」

### 學生常見卡關

| 卡關 | 解法 |
|------|------|
| NotebookLM 產出失敗 | 等 1–2 分鐘後再試，或檢查網路連線 |
| 簡報語言不是中文 | 加上「繁體中文」在提示詞中 |
| 不知道要上傳什麼資料 | 提供範例：課本段落、維基百科條目、自己的筆記 |
| 產出結果不滿意 | 可以重新產出，或調整提示詞 |

### 評量方式

繳交一份自選主題的產出（簡報/心智圖/資訊圖表擇一）。

## 第九課：懶人包 #08 安裝 FB 頻道摘要 Skill

### 備課提醒

- 教師課前先跑一遍 #08 懶人包，確認 SKILL.md 可正常安裝
- 準備好 SKILL.md 的內容（可在投影上顯示或發給學生電子檔）

- 學生能測試「FB 摘要」指令正常運作

### 教學流程

**0'–5' 什麼是 Skill？**

> 老師引導語：「今天要裝最後一個懶人包 #08 — FB 頻道摘要 Skill。先問一個問題：你知道手機為什麼可以安裝 App 來增加功能嗎？」

> 老師引導語：「OpenCode 也有一樣的概念。Skill 就是 OpenCode 的 App。裝了 fb-notebooklm Skill 之後，你只要對 OpenCode 說『FB 摘要』，它就會自動幫你把 FB 內容餵給 NotebookLM，產出報告、簡報、圖表。」

**5'–10' 步驟一：確認認證**

> 老師引導語：「安裝前先確認 NotebookLM 連線正常。打開 PowerShell，輸入：」

```powershell
nlm doctor
```

> 老師引導語：「應該顯示 Authentication valid!。如果顯示未認證，執行 nlm login 重新登入。」

**10'–25' 步驟二：安裝 Skill 檔案（最關鍵的步驟）**

> 老師引導語：「現在要安裝 SKILL.md 檔案。這是最關鍵的一步，請仔細跟著我做。」

> 老師引導語：「首先，在 PowerShell 中建立技能資料夾：」

```powershell
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.config\opencode\skills\fb-notebooklm"
```

> 老師引導語：「接下來我們要建立 SKILL.md 檔案。請打開記事本。」

- **方法一（推薦）：** 投影 SKILL.md 內容（或發電子檔），讓學生從懶人包 #08 的「附錄一」複製貼上到記事本，存檔到 `%USERPROFILE%\.config\opencode\skills\fb-notebooklm\SKILL.md`

> 老師引導語：「複製的時候要全部選取，不要漏掉任何部分。存檔時檔名一定要是 SKILL.md，不是 SKILL.md.txt！」

> ⚠️ 提醒學生：「檔案總管可能隱藏了已知檔案類型的副檔名。請在記事本存檔時，將『存檔類型』改為『所有檔案』，檔名輸入 SKILL.md。」

**25'–30' 步驟三：重啟 OpenCode**

> 老師引導語：「安裝了新的 Skill 之後，要完全關閉 OpenCode 再重新打開，它才會載入新的 Skill。」

**30'–42' 步驟四：測試 Skill**

> 老師引導語：「重啟後，對 OpenCode 說一句話：『FB 摘要』」

> 老師引導語：「如果安裝成功，OpenCode 會載入 fb-notebooklm Skill，然後開始問你問題。你就照著它的引導，一步一步操作。」

- FB 內容來源 → 選「手動貼上」，貼上一段文字
- NotebookLM 筆記本 → 選「新建一本」
- 要產出哪些內容 → 選擇至少一項

> 老師引導語：「如果它開始問你問題，表示安裝成功了！」

**42'–45' 懶人包 #08 完成！**

> 老師引導語：「三個懶人包全部完成了！(#00 環境建置、#01 連接 NotebookLM、#08 FB 頻道摘要 Skill) 你們現在擁有一套完整的 AI 輔助學習工具。最後一節課，我們要來做一個整合應用專題！」

### 學生常見卡關

| 卡關 | 解法 |
|------|------|
| SKILL.md 存成 .txt 檔 | 在記事本存檔時選擇「所有檔案」，檔名打 SKILL.md |
| 找不到技能資料夾 | 路徑是 `%USERPROFILE%\.config\opencode\skills\fb-notebooklm`，注意 `.config` 是隱藏資料夾 |
| 說「FB 摘要」沒反應 | 確認 SKILL.md 在正確路徑，重啟 OpenCode |
| SKILL.md 內容不全 | 對照懶人包 #08 的附錄一，確認內容完整 |
| nlm doctor 未認證 | 執行 nlm login 重新登入 |

### 評量方式

老師巡視確認 SKILL.md 已安裝，且「FB 摘要」指令可正常啟動互動流程。

**❌ 不可以：**
- 直接複製 AI 的回答交作業（抄襲）
- 把個人資料給 AI
- 完全相信 AI 說的每句話
- 用 AI 做傷害別人的事

> 老師引導語：「AI 是很強大的工具，但工具再厲害，使用工具的人才是關鍵。記得三件事：第一，AI 會出錯，要自己判斷；第二，AI 輔助你學習，不是代替你學習；第三，你的腦袋才是最重要的。」

> 老師引導語：「這十節課你們學會了：安裝 OpenCode、使用 NotebookLM、安裝 Skill、整合應用。你們已經比很多大人還了解 AI 工具的用法了！希望你們以後能好好運用這些工具，讓學習更有效率。」

### 學生常見卡關

| 卡關 | 解法 |
|------|------|
| 學生不知道選什麼主題 | 提供範例：社會課某單元、自然課實驗報告、閱讀心得、興趣探索（如NBA球隊分析） |
| 時間不夠完成 | 讓學生說明規劃，承諾課後完成繳交 |
| 學生不敢上台 | 可選擇書面繳交即可 |

## 附錄 C：常見教室技術問題

| 問題 | 解決方案 |
