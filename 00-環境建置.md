# OpenCode 懶人包 #00：環境建置

> 版本：v0.6
> 更新日期：2026-06-11


> 📌 **本懶人包可獨立執行**：會自動檢查並安裝所需工具，不需要先看過其他懶人包。你只要確認下方「先備條件」即可開始。

---

## 這個懶人包會幫你做什麼？

幫你把使用 OpenCode 懶人包所需的所有環境一次建置完成，包括：
- 安裝 Git for Windows（如果是 Windows 使用者）
- 安裝 GitHub CLI（gh）、uv 等基礎工具
- 驗證所有工具都能正常使用

> 💡 **本懶人包只負責安裝工具，不含 GitHub 帳號登入。** 需要連接 GitHub、把教材推上線，請接著執行 **懶人包 #02：連接 GitHub**（內含帳號註冊引導，沒有 GitHub 帳號也能從那裡開始）。

---

## 先備條件

在使用這個懶人包之前，請確認：

- [ ] 電腦有網路連線
- [ ] Windows 使用者（winget 尚未安裝也沒關係，步驟零之一會協助安裝；macOS/Linux 跳過）

---

## 請 OpenCode 幫我執行以下步驟

> ⚠️ 以下內容是給 OpenCode 讀的操作指令。
> 你只需要把這整份 MD 檔丟給 OpenCode 桌面版的 Code 分頁，它會自動開始執行。
> 遇到需要你手動操作的地方，它會暫停並告訴你該怎麼做。
>
> **所有安裝與設定都在 OpenCode 桌面版內完成，不需要另外打開 PowerShell 或命令提示字元。**
> 如果 OpenCode 桌面版無法執行某個指令，才會引導你到終端機操作。
> 進階使用者也可以直接使用 OpenCode CLI 版本來執行本懶人包。

---

### 步驟零：環境檢查

> 🔁 **重複執行安全檢查**：如果所有工具都已安裝，此步驟直接跳過。

請 OpenCode 在開始前，先自動確認以下所有項目。
如果有任何一項不符合，請先告知使用者問題所在，並引導解決後再繼續。
**不要跳過任何一項檢查，不要假設環境正常。**

1. **確認作業系統**：執行系統指令確認是 Windows / macOS / Linux，後續所有指令請根據實際的作業系統選擇正確版本執行
2. **確認網路連線正常**：嘗試 ping 一個外部網站
3. **檢查 Git 是否已安裝**：執行 `git --version`
4. **檢查 GitHub CLI 是否已安裝**：執行 `gh --version`
5. **檢查 uv 是否已安裝**：執行 `uv --version`
6. **檢查 winget（Windows 限定）**：執行 `winget --version`

> 把未安裝的項目記錄下來，後續步驟中會逐一安裝。
> 告知使用者目前的狀態，例如：「Git 已安裝 ✅、GitHub CLI 未安裝 ❌，接下來會幫你安裝。」
> 如果所有項目都是 ✅，直接跳到步驟四（最終驗證）。

---

### 步驟零之一：安裝 winget（Windows 限定）

> 🔁 **重複執行安全檢查**：如果 winget 已安裝，此步驟直接跳過。
> macOS / Linux 跳過此步驟（winget 為 Windows 限定）。

**Windows：**
- 執行以下 PowerShell 指令，在一個區塊內完成檢查與安裝：
  ```powershell
  $bundle = Get-ChildItem -Path ".\" -Filter "*.msixbundle" | Select-Object -First 1
  if ($bundle) {
      Write-Output "在工作目錄中找到 winget 安裝檔：$($bundle.FullName)，正在為你安裝..."
      try {
          Add-AppxPackage -Path $bundle.FullName -ErrorAction Stop
          Write-Output "安裝完成，請繼續驗證。"
      } catch {
          Write-Output "安裝失敗：$_"
      }
  } else {
      Write-Output "未找到 .msixbundle 檔案"
  }
  ```
- 如果 PowerShell 輸出「未找到 .msixbundle 檔案」或「安裝失敗」：
  - 告知使用者：
    - 「⚠️ 工作目錄中沒有找到 winget 安裝檔（.msixbundle）」
    - 「這個安裝檔超過 200MB，檔案很大，沒有直接包含在懶人包中」
  - 引導以下列其中一種方式取得：
    a. 檢查電腦其他位置（如下載資料夾、隨身碟或網路磁碟機）是否有先前下載或取得的 `.msixbundle` 檔案
    b. 手動從 Microsoft 官方下載安裝：
       - **離線安裝包（官方最新版）**：https://aka.ms/winget-cli
       - **GitHub Releases（備用）**：https://github.com/microsoft/winget-cli/releases
  - 使用者下載 `.msixbundle` 後雙擊執行即可安裝
  - 安裝完成後，請再次執行 `winget --version` 確認安裝成功
- 執行 `winget --version` 確認能正常顯示版本號
- 若仍無法使用，請執行以下指令註冊 winget：
  ```powershell
  Add-AppxPackage -RegisterByFamilyName -MainPackage Microsoft.DesktopAppInstaller_8wekyb3d8bbwe
  ```

### 步驟一：安裝 Git

> 🔁 **重複執行安全檢查**：如果 Git 已安裝（`git --version` 成功），此步驟直接跳過。

**Windows**：
```bash
winget install --id Git.Git --accept-source-agreements --accept-package-agreements
```

**macOS**：
```bash
xcode-select --install
```

**Linux（Ubuntu/Debian）**：
```bash
sudo apt update && sudo apt install git -y
```

安裝完成後確認：
```bash
git --version
```

> 如果 `git` 指令找不到，請重開終端機再試一次。

> ⚠️ **PowerShell 5.1 編碼注意事項**：Windows 預設的 PowerShell 5.1 使用系統 ANSI 編碼（繁體中文 Windows = Code Page 950 / Big5），而非 UTF-8。如果後續透過 GitHub Pages 部署網站，中文字可能顯示亂碼。建議安裝 Git 後一併設定：

**方案 A（推薦，一次解決）：全域設定 Git 統一用 UTF-8**
```bash
git config --global core.autocrlf false
git config --global core.safecrlf false
git config --global i18n.commitEncoding utf-8
git config --global i18n.logOutputEncoding utf-8
```

**方案 D（必做）：每個 HTML 都加 `<meta charset="utf-8">`**
即使檔案是 UTF-8 編碼，沒有這行標籤，瀏覽器也無法正確判斷字元編碼。

---

### 步驟二：安裝 GitHub CLI

> 🔁 **重複執行安全檢查**：如果 GitHub CLI 已安裝（`gh --version` 成功），此步驟直接跳過。

**Windows**：
```bash
winget install --id GitHub.cli --accept-source-agreements --accept-package-agreements
```

**macOS**：
```bash
brew install gh
```

**Linux（Ubuntu/Debian）**：
```bash
(type -p wget >/dev/null || (sudo apt update && sudo apt-get install wget -y)) \
  && sudo mkdir -p -m 755 /etc/apt/keyrings \
  && out=$(mktemp) && wget -nv -O$out https://cli.github.com/packages/githubcli-archive-keyring.gpg \
  && cat $out | sudo tee /etc/apt/keyrings/githubcli-archive-keyring.gpg > /dev/null \
  && sudo chmod go+r /etc/apt/keyrings/githubcli-archive-keyring.gpg \
  && echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null \
  && sudo apt update \
  && sudo apt install gh -y
```

安裝完成後確認：
```bash
gh --version
```

> 如果 `gh` 指令找不到，請重開終端機再試一次。

---

### 步驟三：安裝 uv

> 🔁 **重複執行安全檢查**：如果 uv 已安裝（`uv --version` 成功），此步驟直接跳過。
> uv 是 Python 套件管理工具，後續的懶人包安裝 MCP 工具時會用到。

**Windows（PowerShell）**：
```bash
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

**macOS / Linux**：
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

安裝完成後確認：
```bash
uv --version
```

---

### 步驟四：最終驗證

請逐一確認以下所有工具都能正常運作：

```bash
git --version
gh --version
uv --version
```

**Windows 使用者再加驗：**
```bash
winget --version
```

> 全部通過後，告知使用者：
> 「✅ 環境建置完成！你的電腦已經準備好使用所有 OpenCode 懶人包了。」

## 如果安裝失敗，如何重來

對 OpenCode 說：
「環境建置懶人包執行失敗了，幫我檢查哪裡出問題，重新處理。」

OpenCode 會自動：
1. 重新執行步驟零的環境檢查
2. 找出失敗的項目
3. 針對失敗項目重新安裝或設定

## 常見問題

| 問題 | 解法 |
|------|------|
| brew 指令找不到（macOS） | 先安裝 Homebrew：`/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"` |
| 安裝後指令仍找不到 | 關閉所有終端機視窗，重新開啟後再試 |
| 安裝 Git 失敗（Windows） | 手動下載 Git for Windows 安裝檔：https://git-scm.com/download/win |
| winget 指令不存在（Windows） | 先檢查工作目錄有無 `.msixbundle`（步驟零之一），若無則至 https://aka.ms/winget-cli 下載安裝，或從 Microsoft Store 安裝「App Installer」 |
| 執行 `uv` 指令跳出安全性警告（Windows） | 這是正常的 PowerShell 執行策略限制，選擇「Yes」允許執行即可 |
| 我是 macOS 但沒有 xcode-select | 先從 App Store 安裝 Xcode，或執行 `xcode-select --install` 安裝 Command Line Tools |
| Linux 上 `sudo apt update` 失敗 | 檢查網路連線，或更換 mirror：`sudo sed -i 's/archive.ubuntu.com/free.nchc.org.tw/g' /etc/apt/sources.list` |
| 裝完後 `gh` 指令找不到（macOS） | 確認 Homebrew 安裝路徑有在 PATH 中：`echo 'export PATH="/opt/homebrew/bin:$PATH"' >> ~/.zshrc` |
| 可以只裝特定工具嗎？ | 可以。對 OpenCode 說：「幫我檢查環境，只裝 Git 和 uv」，🔁 檢查會自動跳過已安裝項目 |
| 公司有資訊安全政策，安裝要注意什麼？ | 選擇「手動安裝」模式，或先告知 OpenCode 你的限制，它會避開 winget 改用離線安裝 |
| macOS 上 `brew install gh` 很慢 | 可以改用 `brew upgrade gh` 如果已安裝過，或使用官方 GitHub Releases 手動下載 |
| 如何確認所有工具都是最新版？ | 重新執行此懶人包，🔁 檢查完成後告知你要更新的工具版本 |
